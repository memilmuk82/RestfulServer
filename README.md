# Node.js RESTful Server 연습 코드

## 프로젝트 개요

Node.js 기본 `http` 모듈로 사용자 이름 CRUD와 정적 페이지 제공을 구현한 학습용 REST 서버입니다. 외부 서버 패키지나 데이터베이스 없이 HTML·CSS·브라우저 JavaScript와 JSON API를 함께 제공합니다.

## 라우트

| 메서드 | 경로 | 동작 |
|---|---|---|
| `GET` | `/` | 사용자 관리 화면 HTML |
| `GET` | `/about` | 소개 화면 HTML |
| `GET` | `/users` | 전체 사용자 객체 JSON |
| `GET` | `/restFront.css` | 허용된 CSS 파일 |
| `GET` | `/restFront.js` | 허용된 브라우저 JavaScript 파일 |
| `POST` | `/user` | 사용자 이름 등록 |
| `PUT` | `/user/:id` | 해당 키의 사용자 이름 변경 |
| `DELETE` | `/user/:id` | 해당 키의 사용자 삭제 |

정의되지 않은 경로와 허용 목록에 없는 정적 파일 요청은 `404 NOT FOUND`를 반환합니다.

## 요청 처리와 검증

- POST·PUT 요청 본문을 JSON으로 파싱
- 이름이 문자열인지 확인하고 앞뒤 공백 제거
- 비어 있는 이름은 `400`으로 거부
- 올바르지 않은 JSON은 `400`으로 거부
- 1MiB를 초과하는 요청 본문은 `413`으로 거부
- 파일 시스템 전체를 노출하지 않고 CSS·JavaScript 두 경로만 허용
- 파일 읽기와 예상하지 못한 오류는 서버 오류 응답으로 처리

사용자 ID는 등록 시점의 `Date.now()` 값을 사용합니다.

## 기술 구성

- Node.js 기본 `http`, `fs`, `path` 모듈
- 브라우저 JavaScript
- Axios 브라우저 CDN
- HTML과 CSS
- Node.js 기본 테스트 러너

`package.json`과 외부 서버 의존성은 없습니다. Axios는 버전이 고정되지 않은 외부 CDN URL에서 브라우저가 직접 불러옵니다. 지원 Node.js 버전은 저장소에 명시되어 있지 않습니다.

## 프로젝트 구조

```text
.
├── restServer.js      # HTTP 서버, 라우트와 인메모리 사용자 저장
├── restFront.html     # 사용자 등록·수정·삭제 화면
├── restFront.js       # Axios 기반 브라우저 요청
├── restFront.css      # 링크 스타일
├── about.html         # 소개 화면
└── restServer.test.js # 요청 처리 단위 테스트
```

## 실행

```bash
node restServer.js
```

기본 접속 주소는 `http://localhost:8082/`입니다. `PORT` 환경 변수로 포트를 바꿀 수 있습니다.

```bash
PORT=8080 node restServer.js
```

서버 코드에는 수신 호스트가 지정되어 있지 않으므로 실행 환경에 따라 로컬 외의 네트워크 인터페이스에서도 접근될 수 있습니다. 인증이 없는 학습 서버이므로 신뢰할 수 없는 네트워크에 노출하지 마세요.

## 테스트

```bash
node --test
```

테스트 파일에는 다음 네 가지 테스트가 있습니다.

- 허용되지 않은 파일 경로 차단
- 잘못된 JSON을 `400`으로 처리한 뒤 다음 요청 계속 처리
- 1MiB 초과 요청 본문을 `413`으로 거부
- 사용자 등록 및 목록 조회

현재 검증 환경에서 테스트 명령과 네 개의 개별 테스트가 통과했습니다. 인증, 수정·삭제 의미, 동시 요청, 다중 프로세스와 실제 브라우저 UI는 검증하지 않습니다.

## 데이터 저장 방식

데이터베이스와 파일 저장을 사용하지 않습니다. 사용자 이름은 `restServer.js`의 전역 `users` 객체에만 저장됩니다.

- 모든 접속자가 같은 사용자 목록을 공유합니다.
- 서버를 재시작하면 모든 데이터가 사라집니다.
- 여러 프로세스나 워커로 실행하면 프로세스마다 서로 다른 목록을 갖게 됩니다.
- `/users` 응답으로 현재 저장된 모든 이름을 누구나 조회할 수 있습니다.

## 보안 및 동작 제한

- 사용자 인증과 권한 검사가 없습니다.
- 모든 사용자가 임의의 ID에 대해 수정·삭제 요청을 보낼 수 있습니다.
- 존재하지 않는 ID에 PUT 요청을 보내면 새 항목이 생성됩니다.
- 존재하지 않는 ID를 DELETE해도 `200`과 현재 목록을 반환합니다.
- 같은 밀리초에 등록 요청이 처리되면 `Date.now()` ID가 충돌해 기존 항목을 덮어쓸 수 있습니다.
- 브라우저 코드는 `/users` 응답 전체를 개발자 콘솔에 출력합니다.
- 서버는 요청 메서드와 URL을 콘솔에 기록합니다.
- 저장되는 이름은 런타임 데이터이며 개인정보나 민감정보를 입력하기에 적합하지 않습니다.
- 외부 Axios CDN이 차단되거나 오프라인이면 브라우저 CRUD 기능이 동작하지 않습니다.

## 현재 구현되지 않은 기능

- 인증과 사용자별 권한
- 데이터 영속 저장
- 충돌에 안전한 ID 생성
- 존재하지 않는 사용자에 대한 `404` 처리
- 요청 속도 제한과 운영용 로깅
- 운영 배포 설정

## 저장소 관리 제한

`.gitignore`에는 `.venv/`만 포함되어 있습니다. 향후 환경 설정, 로그, 생성 데이터 또는 다른 로컬 파일을 추가할 경우 비밀값과 런타임 데이터가 커밋되지 않도록 제외 규칙을 먼저 정비해야 합니다.

**분류: D. 코드 보존**

Node.js 기본 모듈만으로 HTTP 라우팅, JSON 본문 검증, 정적 파일 허용 목록과 이름 CRUD를 구현하고 단위 테스트를 갖춘 학습 기록으로서 보존 가치가 있습니다. 다만 인증 없이 모든 사용자가 공유하는 메모리 데이터와 단순한 화면 흐름만 제공해 현재 형태의 웹앱 전시 가치는 낮습니다.

- 사용자 미존재 시 PUT·DELETE 응답 의미 수정
- 충돌에 안전한 ID 생성
- 실제 브라우저 통합 테스트와 사용자 흐름 정리
- 영속 저장 정책과 구현
- 수신 호스트 제한과 운영 배포 방식
- Axios 의존성 버전 고정 또는 로컬 제공
